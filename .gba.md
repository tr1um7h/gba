# GBA (Geektime Bootcamp Agent)

GBA is a CLI tool that wraps the Claude Agent SDK to help developers plan and implement features through structured, AI-assisted workflows.

## Technology Stack

- **Language**: Rust 2024 edition (1.85+)
- **Async Runtime**: Tokio
- **AI Integration**: Claude Agent SDK (`claude-agent-sdk-rs`)
- **Template Engine**: MiniJinja
- **CLI Framework**: Clap
- **WebUI**: Axum (HTTP + WebSocket) + vanilla HTML/CSS/JS
- **Serialization**: Serde (JSON, YAML)
- **Error Handling**: `thiserror` (libraries), `anyhow` (application)
- **Logging**: `tracing` ecosystem

## WebUI Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                      GBA CLI (gba-cli)                      │
├─────────────────────────────────────────────────────────────┤
│  commands/plan.rs  ──►  WebPlanApp  ──►  axum + WebSocket   │
│  commands/run.rs   ──►  WebRunApp   ──►  axum + WebSocket   │
└─────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
                    ┌───────────────────────────┐
                    │   Browser (HTML/CSS/JS)   │
                    │   - plan.html             │
                    │   - run.html              │
                    └───────────────────────────┘
```

**Communication Flow:**
1. CLI spawns axum server with WebSocket endpoint
2. Browser auto-opens via `webbrowser` crate
3. `mpsc` channels bridge WebSocket handlers and worker tasks
4. Protocol uses `tag = "type"` for easy `msg.type` switching in JS

## Development Setup

```bash
# Prerequisites
# - Rust 1.85+ with 2024 edition
# - Git
# - Claude Code CLI (configured)
# - GitHub CLI (gh)

# Clone repository
git clone https://github.com/tyrchen/gba.git
cd gba

# Initialize submodules
git submodule update --init --recursive
```

## Build & Test Commands

```bash
# Build
cargo build
cargo build --release

# Run tests
cargo test

# Format code (requires nightly)
cargo +nightly fmt

# Lint
cargo clippy -- -D warnings

# Install locally
cargo install --path apps/gba-cli

# Run in development
cargo run -p gba-cli -- -v <command>
```

## CLI Commands

| Command | Description |
|---------|-------------|
| `gba init` | Initialize GBA in the current repository |
| `gba plan <slug>` | Start interactive planning for a new feature |
| `gba run <slug>` | Execute a planned feature |
| `gba list` | List all features and their status |
| `gba status <slug>` | Show detailed status for a feature |
| `gba remove <slug>` | Remove a feature and clean up resources |

## Configuration

Project configuration is stored in `.gba/config.yml`:

```yaml
agent:
  permission_mode: auto  # auto | manual | none
git:
  auto_commit: true
  branch_pattern: "feature/{id}-{slug}"
review:
  enabled: true
  provider: codex  # codex | claude
```

## Feature Workflow

1. **Plan** - Interactive WebUI chat to discuss requirements and generate design specs
2. **Execute** - Automated execution of planned phases with commits
3. **Review** - AI-powered code review before PR creation
4. **Verify** - Automated verification against acceptance criteria
5. **PR** - Create pull request with generated description

## Conventions

- Use `snake_case` for functions/variables, `PascalCase` for types
- Error handling: `?` operator, never `unwrap()` in production
- Async: Tokio runtime with actor pattern for subsystems
- Logging: `tracing` macros (`info!`, `debug!`, etc.)
- Testing: Unit tests in same file, integration tests in `tests/`
